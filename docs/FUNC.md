## 二、认证授权模块

### 2.1 功能概述

提供用户身份认证和资源访问授权能力，确保平台安全可控。

### 2.2 功能清单

| 功能点     | 能力描述                                                            | 优先级 |
| ---------- | ------------------------------------------------------------------- | ------ |
| 用户登录   | 支持账号密码登录，返回 JWT Token                                    | P0     |
| Token 刷新 | 支持自动刷新 Token，延长会话                                        | P0     |
| 权限验证   | 验证用户对资源的访问权限                                            | P0     |
| 部门管理   | 维护组织架构，支持多级部门                                          | P1     |
| 角色管理   | 维护系统角色（Admin/KnowledgeAdmin/DepartmentManager/Member/Guest） | P1     |
| 审计日志   | 记录所有敏感操作的操作用户、时间、结果                              | P0     |

### 2.3 数据模型

| 实体       | 字段                                                                       | 说明     |
| ---------- | -------------------------------------------------------------------------- | -------- |
| User       | Id, Username, Email, PasswordHash, DepartmentId, Role                      | 用户     |
| Department | Id, Name, ParentDepartmentId                                               | 部门     |
| AuditLog   | Id, UserId, Action, ResourceType, ResourceId, Result, IpAddress, Timestamp | 审计日志 |

### 2.4 权限矩阵

| 角色              | 文档读 | 文档写 | 文档删 | 图谱读 | 图谱写 | 图谱删 | 系统管理 |
| ----------------- | ------ | ------ | ------ | ------ | ------ | ------ | -------- |
| Admin             | ✅      | ✅      | ✅      | ✅      | ✅      | ✅      | ✅        |
| KnowledgeAdmin    | ✅      | ✅      | ✅      | ✅      | ✅      | ✅      | ❌        |
| DepartmentManager | 本部门 | 本部门 | 本部门 | 本部门 | 本部门 | 本部门 | ❌        |
| DepartmentMember  | 本部门 | ❌      | ❌      | 本部门 | ❌      | ❌      | ❌        |
| Guest             | 公开   | ❌      | ❌      | 公开   | ❌      | ❌      | ❌        |

## 三、文档管理模块

### 3.1 功能概述

管理知识文档的整个生命周期：上传 → 解析 → 分块 → 索引 → 检索。

### 3.2 功能清单

#### 3.2.1 文档上传

| 功能点       | 能力描述                                     | 优先级 |
| ------------ | -------------------------------------------- | ------ |
| 本地上传     | 支持从本地上传 PDF、Word、Markdown、TXT 文件 | P0     |
| 文件大小限制 | 单文件最大 50MB                              | P0     |
| 文件格式校验 | 校验文件类型和扩展名是否匹配                 | P0     |
| 自动标题提取 | 从文件名或文档标题中提取文档标题             | P1     |
| 元数据录入   | 支持录入文档的部门、公开范围、访问级别       | P0     |

#### 3.2.2 文档解析

| 功能点        | 能力描述                           | 优先级 |
| ------------- | ---------------------------------- | ------ |
| PDF 解析      | 提取 PDF 文本内容、页数、目录结构  | P0     |
| Word 解析     | 提取 Word 文本内容、标题层级、表格 | P0     |
| Markdown 解析 | 提取 Markdown 标题层级和内容       | P0     |
| TXT 解析      | 直接读取纯文本内容                 | P0     |
| 表格识别      | 识别文档中的表格结构，保持行列关系 | P1     |
| 图片描述提取  | 提取图片的 Alt 文本或 OCR 结果     | P2     |

#### 3.2.3 文档分块

| 功能点       | 能力描述                                      | 优先级 |
| ------------ | --------------------------------------------- | ------ |
| 语义分块     | 按语义边界（段落、章节）切分，适合结构化文档  | P0     |
| 递归分块     | 按层级结构（标题 → 段落 → 句子）递归切分      | P0     |
| 固定长度分块 | 按固定字符数切分，适合简单文本                | P0     |
| 表格分块     | 将表格作为独立块，保持表格完整性              | P1     |
| MoE 分块     | 根据文档类型自动选择最优分块策略              | P1     |
| 分块大小配置 | 支持配置分块重叠度、分块大小（默认 512 字符） | P0     |

#### 3.2.4 向量索引

| 功能点       | 能力描述                                       | 优先级 |
| ------------ | ---------------------------------------------- | ------ |
| 嵌入生成     | 调用 LLM 生成文档块的向量表示                  | P0     |
| 向量存储     | 存储到 PostgreSQL (pgvector)                   | P0     |
| 增量索引     | 新增/更新文档时仅索引变更部分                  | P0     |
| 批量索引     | 支持批量导入文档并索引                         | P1     |
| 索引重建     | 提供全量重建索引接口                           | P1     |
| 索引状态查询 | 查询文档当前的索引状态（待索引/索引中/已完成） | P0     |

#### 3.2.5 文档管理

| 功能点   | 能力描述                               | 优先级 |
| -------- | -------------------------------------- | ------ |
| 文档列表 | 分页查看文档列表，支持按部门、状态筛选 | P0     |
| 文档详情 | 查看文档内容、解析结果、块数量         | P0     |
| 文档更新 | 更新文档内容和元数据，触发增量索引     | P0     |
| 文档删除 | 软删除文档，可配置是否物理删除向量     | P0     |
| 文档下载 | 下载原始文档文件                       | P1     |

## 四、知识图谱模块

### 4.1 功能概述

提供知识图谱的构建、存储和查询能力，支持结构化知识的组织和推理。

### 4.2 功能清单

#### 4.2.1 实体管理

| 功能点   | 能力描述                           | 优先级 |
| -------- | ---------------------------------- | ------ |
| 创建实体 | 创建图谱节点，指定名称、类型、属性 | P0     |
| 查询实体 | 按类型、名称、属性查询实体         | P0     |
| 更新实体 | 更新实体的名称、类型、属性         | P0     |
| 删除实体 | 软删除实体，同时删除关联关系       | P0     |
| 实体详情 | 查看实体的详细信息和关联关系       | P0     |

#### 4.2.2 关系管理

| 功能点   | 能力描述                           | 优先级 |
| -------- | ---------------------------------- | ------ |
| 创建关系 | 创建两个实体间的关系，指定关系类型 | P0     |
| 查询关系 | 按类型、源实体、目标实体查询关系   | P0     |
| 更新关系 | 更新关系的类型、属性               | P0     |
| 删除关系 | 软删除关系                         | P0     |

#### 4.2.3 图查询

| 功能点   | 能力描述                           | 优先级 |
| -------- | ---------------------------------- | ------ |
| 路径查询 | 查询两个实体间的最短路径或多条路径 | P0     |
| 多跳推理 | 支持 1~N 跳的关系推理查询          | P0     |
| 邻居查询 | 查询某实体的所有直接邻居           | P0     |
| 聚合查询 | 对查询结果进行统计（计数、分组）   | P1     |
| 模糊匹配 | 支持按实体名称模糊搜索             | P1     |

#### 4.2.4 图可视化

| 功能点   | 能力描述                      | 优先级 |
| -------- | ----------------------------- | ------ |
| 子图展示 | 展示指定实体为中心的 N 跳子图 | P2     |
| 路径高亮 | 在图上高亮显示查询路径        | P2     |

## 五、检索问答模块

### 5.1 功能概述

实现 RAG（检索增强生成）管道，支持精准的知识检索和智能问答。

### 5.2 功能清单

#### 5.2.1 向量检索

| 功能点     | 能力描述                                | 优先级 |
| ---------- | --------------------------------------- | ------ |
| 语义检索   | 根据 query 生成向量，检索最相似的文档块 | P0     |
| 关键词检索 | 基于 BM25 的关键词匹配检索              | P0     |
| 混合检索   | 融合向量检索和关键词检索的结果          | P1     |
| 结果重排序 | 对检索结果进行 Rerank 提升精度          | P1     |
| 元数据过滤 | 支持按部门、文档类型、时间范围过滤      | P0     |
| 相似度阈值 | 支持设置最低相似度阈值过滤低质结果      | P0     |

#### 5.2.2 RAG 对话

| 功能点     | 能力描述                            | 优先级 |
| ---------- | ----------------------------------- | ------ |
| 单轮问答   | 用户提问，系统检索后生成回答        | P0     |
| 多轮对话   | 支持对话上下文，保持会话连贯性      | P0     |
| 上下文注入 | 将检索结果作为上下文注入 LLM Prompt | P0     |
| 来源标注   | 在回答中标注文档来源，支持溯源      | P0     |
| 拒答能力   | 当检索结果不足时，模型应拒绝回答    | P1     |
| 对话管理   | 支持创建、切换、删除对话会话        | P0     |

#### 5.2.3 检索优化

| 功能点    | 能力描述                               | 优先级 |
| --------- | -------------------------------------- | ------ |
| TopK 配置 | 支持配置返回结果数量（默认 10）        | P0     |
| 检索加速  | 向量索引优化，支持 10000+ 文档秒级检索 | P0     |
| 缓存加速  | 热门 query 结果缓存                    | P1     |

## 六、Agent 对话模块

### 6.1 功能概述

提供基于 LLM 的智能代理能力，支持复杂任务的分解、工具调用和多 Agent 协作。

### 6.2 功能清单

#### 6.2.1 Agent 基础能力

| 功能点        | 能力描述                                | 优先级 |
| ------------- | --------------------------------------- | ------ |
| 单 Agent 对话 | 单个 Agent 处理用户请求                 | P0     |
| 工具调用      | Agent 可调用外部工具（搜索、计算、API） | P0     |
| 多轮执行      | Agent 可进行多轮 LLM 调用直到任务完成   | P0     |
| 上下文管理    | 自动管理对话上下文，防止 Token 溢出     | P0     |
| 错误处理      | 工具调用失败时支持重试或降级            | P0     |

#### 6.2.2 上下文工程

| 功能点     | 能力描述                                       | 优先级 |
| ---------- | ---------------------------------------------- | ------ |
| 上下文模板 | 定义可复用的上下文模板（系统提示词、初始消息） | P0     |
| 变量替换   | 支持模板变量（如 {{user_name}}）               | P0     |
| 消息裁剪   | 上下文超限时自动裁剪旧消息                     | P0     |
| Token 估算 | 实时估算当前上下文 Token 数量                  | P0     |

#### 6.2.3 Agent 编排

| 功能点         | 能力描述                                  | 优先级 |
| -------------- | ----------------------------------------- | ------ |
| Agent 链       | 多个 Agent 串联执行，上一输出作为下一输入 | P1     |
| Agent 池       | 多个 Agent 并行执行，汇总结果             | P1     |
| 意图路由       | 根据用户意图分发到不同 Agent 处理         | P1     |
| 内置 RAG Agent | 专注文档检索问答的 Agent                  | P0     |
| 内置图谱 Agent | 专注知识图谱查询的 Agent                  | P0     |
| 内置 API Agent | 专注外部 API 调用的 Agent                 | P1     |

#### 6.2.4 工具管理

| 功能点       | 能力描述                                  | 优先级 |
| ------------ | ----------------------------------------- | ------ |
| 工具注册     | 注册自定义工具（名称、描述、参数 Schema） | P0     |
| 工具执行     | 执行 Agent 调用工具的请求                 | P0     |
| 工具结果注入 | 将工具执行结果注入 LLM 继续生成           | P0     |

## 七、工作流编排模块

### 7.1 功能概述

提供业务流程的编排能力，支持复杂业务逻辑的组合和执行。

### 7.2 功能清单

#### 7.2.1 Step（步骤）

| 功能点    | 能力描述                             | 优先级 |
| --------- | ------------------------------------ | ------ |
| 定义 Step | 定义最小执行单元（名称、输入、输出） | P0     |
| 执行 Step | 顺序执行单个 Step                    | P0     |
| Step 结果 | 返回执行结果（成功/失败/数据）       | P0     |

#### 7.2.2 Stage（阶段）

| 功能点     | 能力描述                              | 优先级 |
| ---------- | ------------------------------------- | ------ |
| 串行 Stage | 按顺序执行 Stage 内的多个 Step        | P0     |
| 并行 Stage | 同时执行 Stage 内的多个 Step          | P0     |
| Stage 结果 | 返回 Stage 内所有 Step 的执行结果汇总 | P0     |

#### 7.2.3 Pipeline（流程）

| 功能点        | 能力描述                              | 优先级 |
| ------------- | ------------------------------------- | ------ |
| 定义 Pipeline | 定义完整流程（名称、多个 Stage）      | P0     |
| 执行 Pipeline | 按 Stage 顺序执行 Pipeline            | P0     |
| 流程上下文    | 在 Pipeline 内传递数据（key-value）   | P0     |
| 流程结果      | 返回 Pipeline 内所有 Stage 的执行结果 | P0     |

#### 7.2.4 高级特性

| 功能点   | 能力描述                              | 优先级 |
| -------- | ------------------------------------- | ------ |
| 重试机制 | Step/Stage 失败时支持重试（指数退避） | P0     |
| 分支处理 | 根据条件选择不同的执行路径            | P1     |
| 并行执行 | 支持多个 Stage 并行执行               | P1     |
| 子流程   | Pipeline 内嵌套调用其他 Pipeline      | P2     |
